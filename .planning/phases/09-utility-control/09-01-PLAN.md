# Plan: 09-01 Settings & Reader Actions

## Goal
Establish the foundation for v2.0 by introducing a Settings infrastructure, an About view, automatic tagging, and the ability to delete items directly from the Reader view.

## Requirements
- CFG-01: "About" screen.
- CFG-02: Delete item directly from the Reader View.
- CFG-03: Settings menu to toggle behaviors.
- CFG-04: Auto-tag incoming items by domain.
- CFG-05: Auto-tag incoming items by capture year.

## Tasks

1.  **Shared Preferences Integration:**
    -   Add `shared_preferences` package to `pubspec.yaml`.
    -   Create a `SettingsService` to manage boolean flags: `autoTagDomain` (default: true) and `autoTagYear` (default: true).
    -   Register `SettingsService` in `getIt` locator (`main.dart`).

2.  **Settings & About UI:**
    -   Create `SettingsScreen` (`lib/features/settings/presentation/settings_screen.dart`).
    -   Implement toggles for auto-tagging options.
    -   Create `AboutScreen` (`lib/features/settings/presentation/about_screen.dart`) displaying app name, version (via `package_info_plus`), and a brief mission statement.
    -   Add access to `SettingsScreen` and `AboutScreen` from the main `Drawer` in `ItemListScreen`.

3.  **Reader View Deletion (CFG-02):**
    -   Modify `ReaderScreen` to include a `PopupMenuButton` in the `AppBar` actions.
    -   Add a "Delete" option that prompts for confirmation.
    -   Upon confirmation, call `database.deleteItem(item.id)` and `Navigator.pop(context)`.

4.  **Auto-Tagging Logic (CFG-04, CFG-05):**
    -   Modify `IngestionSummaryScreen._handleSave()` or the core insertion logic.
    -   If `autoTagYear` is enabled:
        -   Determine current year (e.g., `DateTime.now().year.toString()`).
        -   Ensure a Tag with this name exists (create if not).
        -   Assign to the new item.
    -   If `autoTagDomain` is enabled and item has a URL:
        -   Extract hostname (e.g., `Uri.parse(url).host.replaceFirst('www.', '')`).
        -   Ensure a Tag with this name exists.
        -   Assign to the new item.

## Execution Rules
- Prioritize clean integration with existing `AppDatabase` and Drift logic.
- Ensure the drawer navigation is smooth and consistent with existing patterns.
