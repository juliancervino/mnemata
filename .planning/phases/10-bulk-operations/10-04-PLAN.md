# Phase 10-04: UX Refinements & Duplicate Detection

## Goal
Improve user feedback during long-running operations and prevent accidental data duplication.

## Tasks
- [x] **UI-10-13**: Implement a visual loading indicator (animated gif/progress) when receiving/processing URLs before reaching the Ingestion Summary.
- [x] **UI-10-14**: Specifically handle the processing state for archive.ph/today links with visual feedback.
- [x] **CON-10-01**: Implement duplicate URL detection and show a confirmation dialog to the user.

## Implementation Strategy
1.  **Loading Indicator**:
    - Identify the entry point for shared intents and manual URL additions.
    - Show a modal loading overlay or a dedicated transition screen with an animated asset.
2.  **Duplicate Detection**:
    - Query the database by URL before opening the ingestion screen.
    - If a match exists, show an `AlertDialog` with "Already exists. Add anyway?" options.

## Verification
- [ ] Sharing a URL from an external app shows a loading state before the summary appears.
- [ ] Attempting to add a URL that is already in the database triggers a warning dialog.
- [ ] Archive links show appropriate processing feedback.
